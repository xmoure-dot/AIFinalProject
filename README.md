Βοηθός Μελέτης με Τεχνολογία RAG
Εφαρμογή Ανάκτησης Πληροφοριών από Έγγραφα
Εργασία για το Σεμινάριο

1. Εισαγωγή
   
Η παρούσα εργασία αφορά την υλοποίηση ενός συστήματος ανάκτησης πληροφοριών με χρήση τεχνολογίας RAG (Retrieval-Augmented Generation). Ο βασικός σκοπός της εφαρμογής είναι να επιτρέπει στον χρήστη να ανεβάζει έγγραφα (PDF, TXT) και να κάνει ερωτήσεις σε φυσική γλώσσα, λαμβάνοντας απαντήσεις βασισμένες αποκλειστικά στο περιεχόμενο των εγγράφων αυτών.
Η ιδέα προέκυψε από την ανάγκη αποτελεσματικότερης μελέτης — αντί να ψάχνει κανείς χειροκίνητα μέσα σε πολλά αρχεία, μπορεί να ρωτά απευθείας το σύστημα και να παίρνει συγκεκριμένες απαντήσεις με αναφορά στην πηγή.

3. Τεχνολογίες που Χρησιμοποιήθηκαν
   
Για την υλοποίηση επιλέχθηκαν οι παρακάτω τεχνολογίες:

Τεχνολογία	Χρήση
FastAPI	Backend API server
ChromaDB	Αποθήκευση vector embeddings
LangChain	RAG pipeline ορχήστρωση
OpenAI API	Δημιουργία απαντήσεων και embeddings
HTML/CSS/JS	Διεπαφή χρήστη (frontend)


3. Αρχιτεκτονική Συστήματος
   
Το σύστημα χωρίζεται σε τρία βασικά μέρη:
Frontend: Απλή HTML/CSS/JS σελίδα που σερβίρεται από τον FastAPI server
Backend API: RESTful API με FastAPI που διαχειρίζεται τα αιτήματα
RAG Pipeline: Αλυσίδα επεξεργασίας που αναλαμβάνει την ευρετηρίαση και αναζήτηση

Όταν ο χρήστης ανεβάζει ένα έγγραφο, αυτό χωρίζεται σε μικρά τμήματα (chunks), μετατρέπεται σε διανύσματα (embeddings) μέσω του OpenAI API και αποθηκεύεται στη ChromaDB. Όταν γίνεται μια ερώτηση, ανακτώνται τα πιο σχετικά τμήματα και παρέχονται στο γλωσσικό μοντέλο ως πλαίσιο για τη δημιουργία της απάντησης.

4. Λειτουργίες Εφαρμογής
   
Ανέβασμα εγγράφων (PDF, TXT, MD) μέχρι 20 MB
Αυτόματη ευρετηρίαση περιεχομένου
Φυσικής γλώσσας ερωτήσεις και απαντήσεις
Εμφάνιση πηγών για κάθε απάντηση
Διαγραφή εγγράφων από το ευρετήριο
Απλή web διεπαφή χρήστη

6. Συμπέρασμα
   
Η εφαρμογή υλοποιεί με επιτυχία ένα λειτουργικό σύστημα RAG που μπορεί να χρησιμοποιηθεί πρακτικά για μελέτη. Η χρήση του OpenAI API για τα embeddings και τη δημιουργία κειμένου εξασφαλίζει υψηλή ποιότητα αποτελεσμάτων, ενώ το REST API επιτρέπει εύκολη επέκταση στο μέλλον.
Πιθανές βελτιώσεις θα περιλάμβαναν υποστήριξη περισσότερων τύπων αρχείων, σύστημα αυθεντικοποίησης χρηστών και αποθήκευση ιστορικού συνομιλιών.
